# さがせ！たからものを！

小学校1年生向けの3DブラウザゲームMVPです。家の中を歩き、ヒントとミニゲームでコインを集め、CPUより先に宝箱を開けます。

## 起動方法

```bash
npm install
npm run assets
npm run dev
```

ブラウザで表示されたローカルURLを開きます。通常は `http://127.0.0.1:5173/` です。

## ビルド

```bash
npm run build
```

## 操作

- `WASD` / 矢印キー: あるく
- マウスドラッグ: カメラ回転
- `Q`: カメラを左へ
- `E`: 近くのものをしらべる。近くに何もない時はカメラを右へ
- `Space` / `Enter`: しらべる
- スマホ横持ち: 左下の矢印ボタンで移動、画面スワイプでカメラ回転、右下のボタンでしらべる
- ミニゲーム: クリックまたはタップで操作

## MVP内容

- 5エリアの1階建て室内: げんかん、リビング、キッチン、こどもべや、ねしつ
- 壁、家具、簡易コリジョン
- ヒント6個、ミニゲームスポット5個
- ミニゲーム3種: おなじ絵、タイミング、数あつめ
- 宝箱はコイン60枚以上で開封
- ライバルCPUが巡回し、ヒントとコインを集め、条件達成で宝箱へ向かう
- WebAudioによる効果音
- 右下ミニマップ、左ヒント一覧、勝利/敗北画面
- スマホ横持ち向けの疑似ボタン操作とタッチ対応ミニゲーム

## デプロイ

`main` ブランチへpushすると、GitHub Actionsで `npm run build` を実行し、`dist` をGitHub Pagesへデプロイします。

## アセット生成

`npm run assets` は `scripts/generate-assets.mjs` を実行します。

- `OPENAI_API_KEY` がある場合: OpenAI Images APIで `public/assets/textures/*.png` を生成します。
- `OPENAI_API_KEY` がない場合、または生成に失敗した場合: SVGフォールバックを `public/assets/textures/*.svg` に生成します。

ゲーム本体はPNGを先に読み、失敗したらSVGを読みます。どちらも読めない場合でも色マテリアルで動きます。

任意で画像モデルを変える場合:

```bash
OPENAI_IMAGE_MODEL=gpt-image-1 npm run assets
```

PowerShellでは次のように指定できます。

```powershell
$env:OPENAI_IMAGE_MODEL = "gpt-image-1"
npm run assets
```

## 既知の制限

- CPUの移動は簡易的な直線移動と少数の中継点です。家具の密集地点では遠回りが不自然に見えることがあります。
- 宝の位置は固定です。遊び直しの変化はまだありません。
- ミニゲームはHTMLオーバーレイ中心で、3D空間内の演出とはまだ完全には連動していません。
- 画像生成PNGがない環境ではSVGフォールバックを使います。実生成テクスチャを入れると見た目の密度が上がります。
